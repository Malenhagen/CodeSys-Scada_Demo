

# CodeSys-Scada_Demo
Данный проект разработан в учебных целях для демонстрации навыков проектирования систем автоматизации, программирования ПЛК и разработки человеко-машинного интерфейса (HMI/SCADA). Система представляет собой программно-аппаратный комплекс управления тремя насосными агрегатами с контурами защит, ведением архивов, гибкой параметризацией и удаленным уведомлением.

Все компоненты комплекса развернуты и эмулируются виртуально на базе SoftPLC, что позволяет демонстрировать проект без привязки к физическому железу.

Технологический стек и архитектура
  
  Нижний уровень (ПЛК): 
    CODESYS V3.5 (на базе виртуального ПЛК CODESYS Control Win V3 x64). 
    Языки программирования: ST (Structured Text).
    Среда передачи данных: Промышленный протокол OPC-UA. Настроен прямой бесшовный обмен тегами между ПЛК и диспетчерским уровнем учитывая права доступа на запись и чтение.
  
  Верхний уровень (HMI/SCADA): 
    Simple-Scada 2 (интуитивный динамический интерфейс, разработанный с нуля).
    Каналы связи и уведомлений: Интеграция с Telegram API через прокси-сервер для оперативного информирования обслуживающего персонала.
  
Схема информационного обмена:[Эмуляция ЩУ (CODESYS)] ──> [Логика ПЛК (ST)] <──(OPC-UA)──> [Simple-Scada 2] ──> [Telegram Bot, архив логов]

  Ключевые инженерные решения, реализованные в проекте
  1. Объектно-ориентированный подход (ООП) в SCADA
     В Simple-Scada разработан кастомный класс PumpClass. Он инкапсулирует:
       - индивидуальное управление системой визуализации состояний агрегата (АВТО, РУЧН СТАРТ, Приостановлен, Отключен, Авария);
       - фильтрацию событий для событийного (а не циклического) логирования.
  
  2. Системные защиты и параметризация из SCADA
     Реализована полноценная подсистема технологических защит. Изменение конфигурации доступно оператору прямо с экрана настроек SCADA:
       - защита от перегрева с динамической уставкой OverTempWarnValue;
       - защита от сухого хода;
       - энергонезависимая память (Persistent/Retain).
     
  3. Профессиональное алармирование и ведение журналов
     Реализована трехуровневая система технологических сообщений:
       INFO — штатные переключения режимов, действия оператора;
       WARN — предупредительные уставки;
       EMERG — аварийные события (перегрев, сухой ход, перелив) с автоматическим остановом технологического процесса.
     Логирование оптимизировано под событийную модель (Event-driven).
     Применена система сообщений об авариях для операторов, со звуковой сигнализацией и требуемым подтверждением сообщения для оценки скорости реакции персонала.

  4. Эмуляция ЩУ и распределение приоритетов управления
    На левом мониторе развернута визуализация CODESYS, полностью имитирующая физический щит управления (ЩУ).
    Реализован переключатель приоритета управления: «Местное (ЩУ) / Дистанционное (СКАДА)».
    При переключении в режим «Местное» верхний уровень управления блокируется графической маской-индикатором, исключая несанкционированное или противоречивое управление процессом со стороны диспетчера, оставляя только информационные данные.

  5. Интеграция с Telegram
    В систему интегрирован Telegram-бот для дублирования аварийных сообщений уровня EMERG на мобильные устройства сотрудников. Предусмотрена возможность включения/отключения уведомлений через интерфейс настроек SCADA. Отправка пакетов настроена через прокси-сервер.

Как запустить проект локально:
ПЛК: Открыть файл demo1CodesysIntegrationLight.project в среде CODESYS V3.5. Запустить эмулятор CODESYS Control Win V3 x64, выполнить Login с загрузкой кода и перевести ПЛК в режим RUN.
SCADA: Скопировать папку проекта SCADA_Simple/ в директорию проектов Simple-Scada (например, C:\Simple-Scada\Projects\). Открыть Project Manager, выбрать проект и запустить Клиент. Проект настроен на локальный адрес 127.0.0.1 по протоколу OPC-UA.
